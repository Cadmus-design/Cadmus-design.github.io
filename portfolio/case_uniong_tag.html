  • 文件標籤系統功能前後台開發

    2021/7

    前後台 wireframe密碼：uniong

  • 依據客服網站重要公告同步教學個案作法，能在教學文件清單頁有對應的依循規則。

  • 需求分析

    1. 教學文件頁面切換事件數量比例偏低，僅佔網站總事件1％
    2. 教學文件頁面瀏覽時間低於5秒的情況明顯佔多數，此行為需進行改善增加瀏覽時間以及減少低於5秒的比例。
    3. 重要公告屬於特殊繁瑣設定以及跨功能操作的指引，但文件是以功能切分的對應方式歸納，故重要公告集結所有內容的方式無法有依循準則提供使用者了解，故需要解決教學文件統整重要公告文件的情況。

  • 產品功能企劃

    1. 建立教學文件關聯標籤系統，增加關聯性，提供更多關聯內容參考，減少問題發生量。
    2. 標籤系統正規化欄位設計，利於內部管理與設定標籤。
    3. 移除上下則切換教學文件功能，簡化資訊內容，並以標籤版位取代，能提供更多指引順序。
    4. 跨功能重要公告可依據功能切分，並增加對應標籤內容，串連整份重要公告切分內容，提供完整系列的操作內容教學。

  • 功能特點

    1. 切分時程先以前台上線為優先，先以前台調整加速提供更多關聯教學文件內容參考。
    2. 簡易直覺的標籤設定方式
    3. 多組檢核機制，確保資訊正確。
    4. 連動式設定，可由標籤管理功能設定套用的教學文件。
    5. 單一教學文件可設定多組標籤對應，提供關聯性內容指引。

    2. 標籤系統後台規劃

  • 覆盤

  • 原因說明：
    經營過wordpress網站經驗，標籤內容會形成浮濫使用、定義不明的情況，不但無法利用標籤系統協助進行關聯設定，也會造成管理困難。

    改善方向：
    規劃後台時不採用搜尋功能能夠簡易分辨已套用或待選標籤是依據上述規則，而標籤規則與管理人討論的決議未落實文件規範，缺少傳承的說明內容，需另外利用時間補充標籤使用規則與方式，改善方向可於決議產生後進行文件規格/規則的落實，避免規則無依循的情況發生。
